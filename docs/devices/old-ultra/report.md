### Display

- `GET /app.json` — active theme. Returns `{"theme": N}` only — no `brt` or `img` fields (see Compatibility). Theme mapping: 1=Weather Clock Today, 2=Weather Forecast, 3=Photo Album (custom image), 4=Time Style 1, 5=Time Style 2, 6=Time Style 3, 7=Simple Weather Clock.

- `GET /timebrt.json` — night mode config. Returns `{"en": 0, "t1": 22, "t2": 7, "b2": 20}` — enable (0/1), start hour, end hour, night brightness level.

- `GET /set?theme={n}` — switch active theme. **State-changing**.

## Things to verify manually

2. **Brightness range -10 to 0** — the firmware slider allows `min="-10"`. This may represent a "backlight fully off" or "very dim" mode. The integration clamps at 0. If a user needs minimum brightness on this device, the true minimum is -10, not 0.

3. **`/set?reboot=1` on Ultra** — the code comments say "Pro devices" but the Ultra JS includes a reboot button and `reboot` appears in the observed parameter list. Likely works on Ultra too; needs confirmation on real hardware.

6. **`/set?fkey={n}`** — appears in the parameter list alongside the OpenWeatherMap key. Could be a "forecast key" type, weather API endpoint selection, or something else. Semantics unknown.
